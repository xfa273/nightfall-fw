# マイクロマウス記事移行計画（はてな -> `nightfall-fw`）

## ゴール

- マイクロマウス関連記事を `docs/blog/posts/` に集約する
- 新規記事をMarkdownで継続執筆できる運用を確立する
- 既存記事は優先度順に移行し、追記・改訂を可能にする

## スコープ

- 対象: マイクロマウス関連の記事
- 非対象: 雑記・無関係記事（必要時のみ別途判断）

## フェーズ

## Phase 1: 運用基盤（今回）

- [x] `docs/blog/` の基本構成を作成
- [x] 記事テンプレートを作成
- [x] 移行トラッカーを作成
- [x] 試し記事を `nightfall-fw` 側へ移設

## Phase 2: 既存記事の棚卸し

- [ ] はてな記事からマイクロマウス関連を一覧化
- [ ] `MIGRATION_TRACKER.md` にURL・公開日・優先度を整理
- [ ] 「そのまま移行」か「改稿して移行」かを判定

## Phase 3: 移行実装（主に私が担当）

- [ ] 1記事ずつ `docs/blog/posts/YYYY-MM-DD_slug/README.md` へ移行
- [ ] 画像を `resources/` へ保存し、リンクを修正
- [ ] 必要に応じて見出し構造・表記ゆれを整える
- [ ] `docs/blog/README.md` の記事一覧を更新

## Phase 4: 公開最適化（任意）

- [ ] GitHub Pages化の要否を判断
- [ ] 必要ならサイト化（検索性・カテゴリ導線・目次）を追加

## 移行1本あたりの手順

1. 対象記事を `MIGRATION_TRACKER.md` で `todo -> doing` に変更
2. 新規フォルダ `docs/blog/posts/YYYY-MM-DD_slug/` を作成
3. 本文を `README.md` に配置
4. 画像を `resources/` へ配置して相対リンクを確認
5. `docs/blog/README.md` の記事一覧へ追加
6. `MIGRATION_TRACKER.md` を `done` に更新

## 分担方針

- あなた: テーマ選定、技術内容の最終判断、公開可否判断
- 私: 下書き作成、過去記事移行、体裁統一、リンク更新

## 品質チェック（公開前）

- 見出し階層が崩れていない
- 画像リンク切れがない
- 文章内の数値・型番・設定値の誤記がない
- 関連するコード・資料へのリンクがある
